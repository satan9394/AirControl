这个 demo 可以定义为一个 **「空中手势桌面控制系统」**，用普通摄像头+手势识别实现鼠标移动、单击、双击、右键、拖拽、滚动、缩放等常用操作，并预留扩展接口。下面是一份可以直接丢进 AI IDE 开干的「项目需求文档 + 技术栈设计」。

## 1. 项目概述

**项目名称**：AirControl – 空中手势桌面控制 Demo
**目标**：
通过电脑摄像头实时识别人类手部姿态，用户无需接触鼠标，仅靠「空中手势」即可操控电脑桌面，实现如下能力：

* 光标移动
* 单击 / 双击
* 右键点击
* 拖拽
* 滚轮上下滚动
* 双指缩放（配合指定应用，比如图片查看）
* 快捷手势（显示菜单、模式切换、校准等）

**运行环境（建议）**：

* OS：Windows 10/11（后续可扩 Linux/macOS）
* 硬件：普通笔记本 / USB 摄像头（≥720p，30fps）
* 主要语言：Python（开发速度快、CV 生态成熟）

## 2. 功能需求

### 2.1 基础功能

1. **手部检测与关键点识别**

   * 支持单手优先，后续可扩展双手
   * 输出至少 21 个关键点（指尖、指关节、手腕）

2. **空中光标移动**

   * 默认使用 **食指指尖位置** 控制光标
   * 支持摄像头空间（像素坐标）到屏幕坐标的映射
   * 支持移动平滑（过滤抖动、插值）

3. **单击**

   * 手势定义示例：

     * 手势 A：食指伸出，拇指靠近食指（pinch），短暂闭合一次
   * 防抖策略：

     * 识别到闭合动作持续 X 帧内，仅触发一次点击事件

4. **双击**

   * 手势定义示例：

     * 在 Y 毫秒内连续两次 pinch
   * 时间窗口参数可配置（如 500ms）

5. **右键点击**

   * 手势定义示例（任选其一）：

     * 食指和中指同时伸出，其余手指收起，快速轻敲动作
     * 或 “反手”手势（手背朝镜头）

6. **拖拽**

   * 手势定义：

     * Pinch 手势保持闭合状态 → 视为按下左键
     * 手指保持 pinch 移动时 → 触发鼠标拖拽
     * 松开 pinch → 释放拖拽（抬起左键）

7. **滚动（滚轮）**

   * 手势候选：

     * 食指与中指并拢向上/向下滑动（两指竖直移动）
     * 或 握拳状态上下移动
   * 上下位移映射为滚轮滚动的增量

8. **缩放（Zoom）**

   * 双手模式（可放到 V2）：左右手各伸出食指和拇指，做开合动作
   * 或 单手两指（拇指+食指）之间距离变化映射为 Zoom in/out

9. **模式切换 / 菜单调用**

   * 例如「张开五指的手掌」持续 1 秒 → 呼出一个屏幕小面板：

     * 开关系统
     * 校准
     * 灵敏度调节
     * 手势说明简图

10. **系统状态提示**

    * 屏幕右下角小浮窗显示：

      * 当前识别到的手势类型
      * 当前模式（光标 / 滚动 / 缩放）
      * 当前帧率

## 3. 手势交互设计（建议全集）

可以按照「姿态 + 动作」组合定义：

| 功能   | 姿态定义（静态）               | 动作定义（动态）            |
| ---- | ---------------------- | ------------------- |
| 移动光标 | 单指（食指）伸出，其余收起          | 无（直接根据位置映射光标）       |
| 单击   | 拇指+食指 pinching         | Pinch 收起持续短时间       |
| 双击   | 同单击                    | 在 500ms 内触发两次 pinch |
| 右键   | 食指+中指伸出，其余收起           | 快速小幅度上下 “点一下”       |
| 拖拽开始 | 拇指+食指 pinch 后保持        | 持续闭合超过阈值时间          |
| 拖拽中  | 维持 pinch               | 移动手指 = 拖拽           |
| 拖拽结束 | 取消 pinch               | 触发鼠标释放              |
| 滚动   | 握拳或两指并拢                | 上下移动映射为滚轮           |
| 缩放   | 双手或单手双指开合              | 指间距离增大/减小           |
| 模式菜单 | 张开五指，手掌对着摄像头           | 保持姿势 1 秒            |
| 校准   | 比 “OK” 手势（拇指+食指圈，其余伸出） | 保持 1 秒，记录当前位置为参考    |

## 4. 系统架构设计

### 4.1 模块划分

1. **Camera Capture（摄像头采集层）**

   * 使用 OpenCV 打开摄像头（cv2.VideoCapture）
   * 输出：原始帧（BGR / RGB）

2. **Hand Tracking Engine（手部追踪引擎）**

   * 利用 MediaPipe Hands / OpenCV + 深度学习模型
   * 输入：图像帧
   * 输出：每只手 21 关键点坐标 + 置信度

3. **Gesture Interpreter（手势识别模块）**

   * 输入：关键点坐标序列（当前帧 + 历史若干帧）
   * 内部步骤：

     * 关键点归一化（相对手掌大小、旋转校正）
     * 手指是否伸直/弯曲的判定
     * Pinch/抓握等形状判定
     * 动作识别（利用时间窗口，判断点按/滑动/连击）
   * 输出：高层手势事件（Move, Click, DoubleClick, DragStart, DragEnd, ScrollUp, ScrollDown, ZoomIn, ZoomOut, Menu, Calibrate 等）

4. **Coordinate Mapper（坐标映射模块）**

   * 输入：手指关键点坐标（相机坐标系）
   * 功能：

     * 去畸变 & 裁剪 ROI
     * 将手部区域映射到屏幕分辨率（如 1920x1080）
     * 设置「活动区域」和「死区」（防止边缘误触）
     * 使用滑动平均或一阶低通滤波去抖

5. **OS Control Adapter（操作系统控制适配层）**

   * 对接系统 API 模拟鼠标/键盘：

     * Python：pyautogui / pynput
   * 输入：手势事件 + 屏幕坐标
   * 输出：真正的 OS 级事件：

     * moveTo(x,y)
     * click()
     * doubleClick()
     * rightClick()
     * scroll()
     * mouseDown() / mouseUp()

6. **UI Layer（可视化与配置界面）**

   * 使用 PyQt5/PySide6 或 Tkinter 简单实现：

     * 设置页面：选择摄像头、灵敏度、手势开关
     * 状态浮窗：显示当前手势、FPS、事件 log
     * 可选：显示摄像头画面+骨架，可用于调试

7. **Config & Logging 模块**

   * 读取/写入配置（JSON/YAML）
   * 手势分析 log（方便后续做模型训练）

## 5. 技术栈建议

### 5.1 语言 & 基础库

* **Python 3.10+**
* **OpenCV**：摄像头读取、基础图像处理、调试可视化
* **MediaPipe Hands**（Google）

  * 优点：轻量、封装好手部关键点检测，适合 demo
  * 替代/扩展：自己的 ONNX 模型 + ONNX Runtime

### 5.2 手势识别方案

1. **第一版：规则引擎 + 几何特征**

   * 对关键点做如下特征：

     * 每个指尖到手腕的向量
     * 指节之间夹角
     * 指尖之间距离（如拇指-食指）
   * 用规则判断：

     * 距离 < 阈值 → pinch
     * 指尖 y 坐标 / x 坐标关系 → 上下、左右动作
   * 优点：实现快、可在 AI IDE 里面快速迭代

2. **第二版：轻量 ML 分类器（可选）**

   * 收集若干帧数据，标注为「单击手势」「右键手势」…
   * 用简单模型训练：

     * MLP / RandomForest / SVM，对输入特征为 21×2 关键点（归一化后）
   * 用模型输出当前帧的「手势类别」，再配合时间窗口判断动作（单次/连续）

### 5.3 系统控制

* **pyautogui** 或 **pynput**

  * 控制鼠标移动、单击、双击、右键、滚轮
* Windows 下如需更低延迟，可考虑直接调用 Win32 API（pywin32）

### 5.4 桌面 UI & 浮窗

* **PySide6 / PyQt5**

  * 制作一个无边框、透明背景的悬浮窗
  * 显示当前手势图标 / 文本说明 / FPS
* 或者简化：仅在 OpenCV 窗口中 overlay 提示

## 6. 非功能性需求

1. **实时性**

   * 目标：普通笔记本上保持 15–25 FPS
   * 延迟：从手势到鼠标响应 < 150ms

2. **鲁棒性**

   * 对光照变化、肤色、背景复杂度有一定容忍度
   * 不需要「极度鲁棒」，但要有基本策略：

     * 检测不到手时暂停发送鼠标事件
     * 手离开画面自动「失活」

3. **安全性**

   * 所有处理在本地完成，不上传图像
   * 可在 UI 中明确标注「本地推理」

4. **可扩展性**

   * 手势表可以配置 / 插件化：

     * JSON 定义：哪个手势映射到哪个系统动作
   * 为之后接入 LLM 做「自定义手势配置解释器」留接口（你以后玩）
