> ### 二、总体开发要求
>
> 1. **开发语言与环境：**
>
>    * 使用 Python 3.x。
>    * 依赖尽量控制在常见库：如 `opencv-python`、`mediapipe`、`pyautogui` 或 `pynput`、`PySide6/PyQt5` 等。
>    * 所有依赖写入 `requirements.txt`。
>
> 2. **工程结构要求（示例，可根据需要微调）：**
>
>    * `aircontrol/`
>
>      * `__init__.py`
>      * `camera.py`         # 摄像头采集
>      * `hand_tracker.py`   # 手部关键点检测封装（如基于 MediaPipe）
>      * `gesture_engine.py` # 手势识别与状态机
>      * `mapper.py`         # 相机坐标 → 屏幕坐标映射 & 平滑
>      * `os_controller.py`  # 调用系统鼠标控制（pyautogui / pynput）
>      * `ui.py`             # 浮窗 / 配置界面（如使用 PySide6）
>      * `config.py`         # 参数配置（灵敏度等）
>    * `main.py`             # 入口，整合各模块
>    * `requirements.txt`
>    * `README.md`
>
>    请根据需求文档，把功能合理拆分到上述模块（如有更合理的拆分，可以调整，但要保持清晰分层）。
>
> 3. **实现优先级：**
>    优先保证以下能力可以工作（可以先做「无 UI 简化版」，再加 UI）：
>
>    1. 打开摄像头，识别手并画出关键点。
>    2. 用食指指尖控制屏幕光标的移动（含坐标映射与平滑）。
>    3. 识别以下核心手势并转为系统鼠标事件：
>
>       * 单击（pinch 一次）
>       * 双击（时间窗口内两次 pinch）
>       * 右键
>       * 拖拽：pinch 按下 → 移动 → 松开
>       * 滚轮上下（握拳或两指上下滑动）
>    4. 提供一个简单状态输出方式：
>
>       * 控制台打印当前识别到的手势类型、坐标等；
>       * 如有余力，再实现一个简单浮窗 UI 显示当前模式 / FPS。
>
> 4. **代码风格要求：**
>
>    * 使用面向对象设计：如 `HandTracker`、`GestureEngine`、`MouseController` 等类。
>    * 每个核心类都写清楚：初始化参数、主要方法、输入输出。
>    * 关键算法（手势判断、平滑、状态机）要有注释解释逻辑。
>    * 所有硬编码阈值（距离阈值、时间窗口、平滑参数等）放在统一配置处，方便调整。
>
> 5. **健壮性：**
>
>    * 未检测到手时，不要乱动鼠标，应自动进入「空闲状态」。
>    * 要考虑帧率与延迟，避免在循环中做重度阻塞操作。
>    * 出错（如摄像头打开失败）时，给出清晰报错信息。
>

> ### 三、开发步骤（请按步骤向我展示过程）
>
> 请按以下顺序开发，并在每一步产出**对应的代码与说明**，而不是一次性给出全部代码：
>
> 1. **步骤 1：工程初始化**
>
>    * 设计/确认项目目录结构。
>    * 给出 `requirements.txt` 初版。
>    * 给出 `main.py` 的最简骨架（只包含 main 循环和几个 TODO）。
> 2. **步骤 2：摄像头与手部检测模块**
>
>    * 实现 `camera.py` 与 `hand_tracker.py`：
>
>      * 能打开摄像头，获取帧。
>      * 使用 MediaPipe Hands（或等价方案）输出手部关键点。
>      * 在窗口上把手骨架画出来。
> 3. **步骤 3：坐标映射与光标移动**
>
>    * 实现 `mapper.py` 和 `os_controller.py` 基本功能：
>
>      * 从食指指尖得到屏幕坐标。
>      * 实现平滑处理（简单滑动平均即可）。
>      * 移动系统鼠标并验证延迟。
> 4. **步骤 4：基础手势（点击 / 双击 / 拖拽 / 滚动）**
>
>    * 实现 `gesture_engine.py`：
>
>      * 利用关键点和历史帧判断：pinch、握拳、两指滑动等。
>      * 用时间窗口识别单击 / 双击。
>      * 定义简单状态机用于拖拽开始 / 结束。
> 5. **步骤 5：整合与简单 UI**
>
>    * 在 `main.py` 中整合：
>
>      * Camera → HandTracker → GestureEngine → Mapper → OSController。
>      * 至少用控制台输出当前手势。
>      * 如有时间，再实现一个简单 UI 浮窗用于显示状态（可选）。
